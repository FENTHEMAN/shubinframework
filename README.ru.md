# Shubin Framework

*Context-first markdown-фреймворк для маленьких команд, работающих с AI-агентами.*

**Языки:** [English](./README.md) · Русский (этот файл)

---

## Что это

Набор практик работы с markdown-vault как **общим контекстом команды** для людей и AI-агентов (Claude Code, Cowork).

Когда команда использует AI-агентов в ежедневной разработке, появляются две боли:

- **Амнезия AI-агентов** — каждая сессия с нуля, решения вчерашнего дня приходится объяснять заново.
- **Потеря контекста людьми** — через три месяца никто не помнит, почему API возвращает camelCase, или почему онбординг в 4 шага, а не в 1.

Это одна и та же проблема: *контекст команды живёт в головах, а не в читаемом виде.*

Shubin Framework выносит контекст в один markdown-репозиторий (vault), который одинаково читают и люди, и AI-агенты. Плюс — опциональный **knowledge-модуль**, реализующий метод Андрея Карпати «LLM wiki», где LLM активно поддерживает compiled research-страницы из raw-источников.

## Для кого

- Команды 2–8 человек, строящие продукт от MVP до early traction
- Использующие AI-агентов (Claude Code, Cowork) как полноценных членов команды
- Имеющие хотя бы одного человека, готового **продавливать практики** (tech lead, founder)

**Не для:** команд 15+, команд без driver'а дисциплины, агентств с flow клиентских проектов, и чисто solo-работы, где полная версия избыточна (см. lite-версию).

## Концепция vault

```
vault/
├── stable/              меняется редко (stack, team, glossary)
├── active/              активная работа (features, decisions/ADR, patterns)
├── log/                 append-only (feedback, incidents, ретро)
└── knowledge/           опциональный Karpathy LLM-wiki модуль
    ├── raw/             immutable внешние источники
    ├── wiki/            LLM-compiled страницы с cross-refs
    └── CLAUDE.md        инструкция LLM как maintainer
```

Два принципа, на которых стоит весь фреймворк:

1. **Append-only для истории, mutable для состояния.** Решения, фидбек, incidents — только дописываются. Roadmap, feature-страницы, статусы ADR — меняются.
2. **Одна правда на каждый факт.** У каждого факта ровно одно каноническое место жительства. Дубли расходятся.

Всё остальное — следствия.

## Связь с методом Карпати

В апреле 2026 Андрей Карпати опубликовал паттерн «LLM wiki» — трёхслойную базу знаний (raw immutable источники → LLM-compiled wiki с cross-references → CLAUDE.md схема), где LLM является maintainer'ом.

Ядро Shubin Framework (главы 2–11 документа `framework.md`) — это классическая engineering-практика: ADR, feature specs, паттерны, runbooks — люди пишут, LLM читает. Метод Карпати реализован как отдельный опциональный модуль (`knowledge.md`), где LLM становится maintainer'ом wiki. См. документ для полной механики.

Оригинальный gist Карпати: https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f

## Честные предупреждения

Фреймворк окупается на 2–3 месяце, не сразу. Без человека, продавливающего практики, умирает за квартал. Не всё здесь подходит каждой команде — адаптируйте, а не копируйте.

Если через 3 месяца vault не используется — это обычно не провал фреймворка, а сигнал о более глубокой проблеме: команде не хватает ясности в том, что вы строите. Никакой инструмент эту ясность не заменит.
